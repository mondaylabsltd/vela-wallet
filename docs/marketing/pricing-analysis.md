## 一、先回答战略问题:付费下载在这门生意里买的是什么

【事实】收入结构 = 商店一次性买断 + relayer fee(≈1× 网络 gas,whitepaper/FAQ)。
【推理】relayer 单用户年收入很低:L2 上单笔 gas 常在几美分量级,活跃用户一年几十笔 ≈ 几美元;主网重度用户才显著。**一次 $39.99 的买断 ≈ 一个 L2 活跃用户十年量级的 relayer 贡献** —— 付费下载不是小补贴,它实质性改变单位经济,尤其对"装了不交易"的用户(relayer 模型下价值 ¥0,买断模型下价值 = 售价)。

付费下载同时承担三个角色:
1. **收入**:直接、可预测、与交易频率解耦
2. **信任信号**:"你付钱,所以你是客户,不是产品"——与"无代币、不割你"(lead #9)、"诚实 alpha"(lead #15)同一叙事轴。免费钱包的隐含问题"它靠什么赚钱?是不是要发币割我/卖我数据?"被定价本身回答了
3. **用户过滤器**:愿付 $10 的人更可能是真的把钱放进来的人(变现核心 Persona A),而非撸空投的一次性安装

## 二、市场锚点(已核实)

| 锚点 | 价格 | 含义 |
|---|---|---|
| 主流钱包 App(Trust/Exodus/Coinbase/MetaMask 等) | **全部免费下载** | 付费钱包=品类异类。风险:商店比价即死;机会:**"唯一收费的钱包"本身是差异化故事** |
| 硬件钱包(冷钱包) | **$25–$500 一次性** | 加密用户已被教育"为私钥安全一次性付费是正常的"——这是 WTP 存在的最强证据 |
| Procreate(iPad 创作工具,买断制标杆) | **$12.99 一次性,升级永久免费** | 工具类买断价格天花板参考;"pay once, updates forever" 的用户好感模型 |
| Things 3(任务管理,买断) | **iPhone $9.99 / iPad $19.99** | 高质感个人工具的标准买断价位 |
| 商店抽成 | **双店均 15%**(Apple Small Business Program 需申请;Google 首 $1M 自动) | 净收入 = 标价 × ~85% × (1 − 税) |

品类内无直接可比付费钱包(搜索未发现主流案例)——定价对标只能取"硬件钱包(上界心理锚)× 精品工具 App(实际价位带)"。

| Persona | 对价格的反应 | 结论 |
|---|---|---|
| **A 多链 DeFi 重度玩家**(变现核心) | 管理数千至数十万美元资产;被盲签/无限授权坑过;本来就在考虑 $59–$249 的硬件钱包。$39.99 vs "一次错误授权归零"仍不构成决策变量。**付费敏感度极低,信任敏感度极高** | 定价对 A 几乎无阻力,是买断收入主力。文案打 #4/#17/#16(拒绝盲签/永不无限/余额模拟)+ 硬件钱包对标 |
| **B 被助记词坑过的转化者**(获客主力) | 有真实损失记忆,WTP 存在但"新钱包凭什么信你"异议在先。**先信任后付费:Web 免费试用是必经路径** | Web→付费转化的核心人群;商店直接冷转化难 |
| **C 隐私/主权极客** | 原则上认同"付费=激励对齐"(反数据变现、反 token 补贴);但要求开源无阉割。**付费本身可以是打动 C 的论点** | 布道者:让 C 转发"为什么我们收费"宣言 |
| **D 新手/大众** | 价格敏感,画像里"③免费"本来就在其在意清单——**D 不是移动付费版的目标**,由免费 Web 版承接 | 商店listing被 D 看到→跳出是预期内损耗,靠 Web 漏斗兜住 |
| **E 开发者/BD** | 不看价格看代码 | 无影响 |

**结构性结论**【推理】:付费下载天然把商店版定位给 A(+部分 B/C),Web 免费版承接 B/D 获客——这与既有的"两层营销"战略(获客入口≠变现核心)完全同构,不冲突,反而更清晰。

## 四、定价建议

### 模式:付费买断(paid upfront),不做订阅、不做免费+IAP 解锁

- **订阅 ❌**:"停止付费钱包就降级"与自托管叙事根本冲突(资金主权不能有月租);加密人群订阅疲劳
- **免费+一次性 IAP 解锁 ❌(暂不)**:保留了商店试用漏斗,但必须决定"锁什么"——锁安全功能(clear-signing/授权上限)直接背叛品牌承诺;锁装饰性功能则说服力弱,还要新增门控代码和测试面。Web 免费版已经是完整试用,不需要第二个试用层
- **付费买断 ✅**:实现成本为零(无门控代码)、叙事最干净("买了就是你的,全部功能,永久更新")、与 Procreate 模型一致
- ⚠️ **单行道警告**:Google Play 付费应用一旦改为免费,**不可改回付费**(Play 长期政策,上架前在 Play Console 文档再核实一次)。所以**宁可首发定价,不要"先免费冲量再收费"**——反向不可逆

### 价格点:**$39.99 美元(双平台同价)** —— 创始人定调"高到不能再高"(2026-07-02)后的顶格定价

> 初版分析推荐 $9.99(转化最优解)。创始人明确要求**最大化单价**,策略换轨为"找不崩的天花板"。天花板由两条已核实的硬约束夹出:

**约束一(上界,硬):最便宜的硬件钱包。** 2026 实价:Ledger Nano S Plus ≈ **$44**(€40.83)、Trezor Safe 3 **$59**。软件版贵过它们,理性用户直接买硬件——"比最便宜的硬件钱包还便宜"这条核心叙事就死了。**绝对不可越过 ~$44。**

**约束二(参照,软):消费级买断 App 的历史天花板。** GoodNotes Special Edition **$35.99**、Loopy Pro **$29.99**、Nomad Sculpt ~$15、Things 3 iPad $19.99、Procreate $12.99。$36 是大众买断 App 的实际最高档——**$39.99 = 登顶全品类,同时仍在硬件钱包锚之下**。

| 阶梯 | 评估 |
|---|---|
| $19.99–$29.99 | 稳妥的 Pro 价位(Loopy Pro 档),但不满足"顶格"要求 |
| **$39.99 ✅ 顶格推荐** | 贵过所有主流买断 App(本身即定位声明),低于 Nano S Plus($44)/Safe 3($59);净收 ~$34/单;支撑"签名设备"而非"应用"的定位 |
| $49.99 | ⚠️ 与 Nano S Plus 价格相撞,锚点开始失效;仅当叠加 Founders Edition 稀缺性叙事才勉强成立 |
| ≥$59 | ❌ 贵过 Trezor Safe 3,对任何画像都不理性,不讨论 |

**顶格定价的连带策略(与 $9.99 方案的关键反转)**:
- **首发即 $39.99,没有低价首发期**。买断制 App 降价是单行道声誉事件(早期买家付了 $40,三个月后见 $15 → 一星差评);涨价无人受伤,降价人人受伤。顶格策略下 urgency 工具从"要涨价"换成 **Founders 权益**(首批买家永久 Founders 标识/更新承诺,成本为零)
- **区域定价 + 促销:两层结构**(创始人 2026-07-02 确认:$39.99 为默认价,允许分区域打折促销):
  - **常驻 PPP 区域价(不是促销)**:一线市场(美/欧/英/日/韩/新/港台)固定 $39.99 等值;新兴市场(印尼/越南/土耳其/拉美/巴西,对应既有 vi/id/tr/es-MX/pt-BR 语言包)常驻 $19.99–$24.99 等值。两店均支持逐国覆盖价。购买力定价不伤一线市场价格完整性
  - **限时促销(事件化)**:每年 ≤2 次;一线市场地板价 **$29.99**,新兴市场可更深;Google Play 有原生划线促销价,Apple 靠改价 + 官网/社媒造势
  - **三条护栏**:① 任何地区任何促销**永不设 $0**(Play 付费→免费不可逆);② 首发后 90 天内一线市场不促销(保护 Founders/A 画像);③ 区域差价 ≤~50%,抑制跨区套利动机(换区需换支付方式,摩擦天然够高)
- 收入参照【推理】:$39.99×85%≈$34/单 → $10K MRR 只需 **~295 单/月**($9.99 方案需 1,180 单);100 单/月即 $3.4K MRR 底座。顶格价把"需要说服的人数"降到 1/4,代价是每一单都更依赖信任叙事到位

### 付费版到底"多"什么(定价的诚实依据)

**不做功能阉割**——Web 与原生功能对等是信任资产。付的是:
1. **原生 passkey 体验**:Secure Enclave/StrongBox 直连、系统生物识别、无浏览器层(lead #22 "和 Apple Pay 同款硬件保护"在原生端才是完整故事)
2. **原生体验**:相机扫码、触觉反馈、系统级键盘/分享、离线打开(lead #46 到账触觉这类"惊喜时刻"是原生独有)
3. **供养开发**:"没有代币、没有 VC 补贴叙事、没有数据变现——App 售价就是我们的收入来源之一,这就是激励对齐"(与 whitepaper 诚实风格一致)

## 五、说服付费:转化叙事与漏斗

### 核心文案框架(按画像)

- **对 A**:硬件钱包对标框架——"你已经拥有一块安全芯片(Secure Enclave/StrongBox)。$39.99 把它变成你的签名设备:clear-signing、永不无限授权、签前余额模拟——比最便宜的 Ledger 还便宜,没有第二台要充电、会丢、要藏的设备。"(锚:$59–$249 硬件钱包 + 被盗金额,永远不是"别的 App 多少钱")
- **对 B**:信任递进框架——"先在浏览器免费用,放小额,连 dApp,全部试完。觉得值,再买手机版。"(**Web 免费版就是退款保证的替代品**——'先试后买'比商店退款政策更强)
- **对 C**:激励对齐宣言——"我们为什么收 $39.99:无代币、无广告、无数据变现。你付钱,你就是客户。MIT 开源,不满意可以自己编译。"(注意:开源侧应答"可自建"——付费买的是签名版本+商店分发+自动更新+供养维护,这是坦荡的交换,C 吃这套)
- **通用红线**(承接 leads 红线):永不说"已审计/审计排期";永不写 "free app";"先用小额"当卖点讲

### 关键转化时刻(Web→商店)

在 Web 版做原生升级引导(工程量小,均为既有能力):
1. 首笔交易成功回执页 → "在手机上随时签名:iOS/Android 版"
2. 首次 dApp 连接成功 → "手机扫码连接更顺手"
3. 到账时(Web 无系统推送)→ "原生版支持到账触觉/通知"(若通知未建,勿写,以实际功能为准)
4. getvela.app 官网定价区:三列 Web(免费)/ iOS($39.99)/ Android($39.99),Web 列写 "full features, free forever" —— 免费列的存在本身就是付费列的信任背书;高价下建议第四列放"硬件钱包 $59–$249"做对比锚(诚实标注是第三方产品)

## 六、验证计划(定价不是拍脑袋一次定死)

1. **上架前(顶格价下不可跳过)**:在既有社群/CT 跑 Van Westendorp 四问(太贵/太便宜/偏贵可接受/划算),n≥50,重点看 "too expensive" 拐点是否真在 $39.99 之上;若大面积落在 $20–$30,顶格价要靠 Founders 权益/定位文案补,而不是硬扛
2. **首发期**:$39.99 直接上 → 观察 Web→商店点击转化率、商店页转化率(两店后台都有);若商店页转化 <5%【推理阈值,高价下基线更低】,优先改 listing 文案与"签名设备"定位,而非降价
3. **梯子备份**:若 90 天数据证明 $39.99 不成立,退阶到 $29.99(Loopy Pro 档)——一次性、公告式、给早期买家 Founders 补偿叙事;**只允许退这一阶,不允许多次滑坡**
4. **区域价**:顶格策略下保持全球单一价 ±小幅;新兴市场由免费 Web 版承接,不做深折扣
5. **不可做**:降到免费再收费(Play 单行道);按国家 A/B 同时挂两个价(品牌风险);频繁促销(价格完整性即定位)
