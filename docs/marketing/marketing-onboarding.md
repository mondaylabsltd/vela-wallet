# 市场负责人带训手册(新人接手分发·曝光·转化)

> 2026-07-02。背景:创始人专注产品构建,marketing 移交给一位新人同事。
> 本手册回答三个问题:她按什么顺序成长、哪些事她能独立做、创始人的时间花在哪。
> 设计原则:**这个品牌最大的营销风险是"说错话"而非"做得少"**(honest-alpha 定位,一句"已审计"/"完全免费"的代价远大于三个月不发内容)。所以先事实、后手感、再放权。

---

## ⚠️ 模式修正(2026-07-02 创始人决定):第一个 90 天,创始人主导、同事辅助

[90-day-gates.md](90-day-gates.md) 的生死对表要求 90 天数据反映**市场的真实回答**,而不是新人的学习曲线。因此上架后的第一个 90 天:

| | 创始人(主导) | 同事(辅助) |
|---|---|---|
| **对外发声** | Show HN、Product Hunt maker、CT 线程、Reddit 答题、安全事件评论、播客——**全部公开声音是你** | 起草初稿、整理素材、排期提醒、评论区舆情汇总(不代表官方回复) |
| **生产支持** | 定方向、审事实 | 目录提交、截图/录屏素材库、对比视频剪辑、本地化翻译稿、度量看板与漏斗周表 |
| **研究** | 消费结论 | VOC 挖矿(差评+Reddit 原话词库)、竞品动态周报 |

- 她的带训阶段 1-2(事实底盘+低风险执行)**照常进行**——那些本来就是辅助型任务,与此模式零冲突
- 阶段 3-4(带训发布→放权)**推迟到 90 天对表之后**:GREEN/YELLOW → 按原计划移交;RED → 项目降级,她若兼职保温则直接按阶段 4 的模板化内容跑
- 创始人时间现实:**90 天窗口内产品进入只修 bug 不加功能的冻结期**,你的主要时间给营销(发布周 ~60-70%,引擎期 ~40%)。窗口开始前必须完成全部上架阻塞项(签名/真机验证/商店材料)——门槛文档的计时规则已要求这一点
- 本手册其余章节(红线清单、周会节奏、RACI)在 90 天内照用,只是"审批窗口"反转为**她审你**——她拿红线清单§三检查你的每篇发布(新人执行清单比老手凭感觉更严格,这同时是她最好的训练)

## 〇、给创始人:你的角色和时间预算

| 阶段 | 你每周投入 | 你做什么 |
|---|---|---|
| 第 1-2 周 | ~2h | 一次 kickoff(2h,走一遍产品+四份文档地图);批改她的"一页纸产品复述" |
| 第 3-4 周 | ~1.5h | 固定周会 30min + 审她的 VOC 报告和素材库 |
| 第 5-8 周 | ~1.5h | 周会 30min + 每周一个固定 30min"发布审批窗口"(集中审,不随时被打断) |
| 第 9 周起 | ~1h | 周会 30min + 只审"红线触发项"(见 §三) |

**纪律:除安全事件评论外,不做异步随叫随审。** 所有待审内容进一个共享文档,在审批窗口一次过。保护你的产品时间是这套设计的第一目标。

## 一、四阶段成长路径(90 天)

### 阶段 1(第 1-2 周):建立事实底盘 —— 只输入,不输出

**她做:**
1. **先当用户**:在 getvela.app 创建钱包 → 放 $20 → 发一笔 → 连一个 dApp → 故意触发一次授权上限编辑器。没亲手用过产品的人写不出能活的文案
2. **按序精读四份文档**(这是她的教材,顺序即难度):
   ① [100-marketing-leads.md](100-marketing-leads.md) —— 先读"〇、先读"和"一、潜在客户分析"(画像+红线),100 条先浏览
   ② [pricing-analysis.md](pricing-analysis.md) —— 为什么 $39.99、为什么商店不是获客入口
   ③ [why-we-charge.md](why-we-charge.md) —— 品牌声音的标定样本(以后她写的每篇东西,语气向这篇对齐)
   ④ [distribution-plan.md](distribution-plan.md) —— 她未来的作战地图
   ⑤ [fireworks-vs-engine.md](fireworks-vs-engine.md) —— 烟花与引擎:判断每一份时间投入是"资产还是消息"的底层模型
   辅助:`docs/CONTENT-SOURCE-100-CLUES.md` 是事实库,写任何产品声明前先查它(README 已过期,**不许**从 README 取事实)
3. **毕业作业:一页纸产品复述**——用她自己的话回答:这是什么/给谁/凭什么信/怎么赚钱/绝不说什么。

**你做:** 批改这一页纸。她哪里理解偏了,现在纠正成本最低。这次批改是整个带训里杠杆最高的 2 小时。

**通过标准:** 一页纸里五个问题全对,且能脱稿说出 §三 的全部红线。

### 阶段 2(第 3-4 周):低风险执行 —— 动手,但不公开发声

全部选自 distribution-plan 里**机械性高、判断量低、不对外发言**的任务:

1. **目录铺设**(计划 P1-3):20+ 目录提交,逐个核对收录标准。练习:如实填表就是练"只说事实库里有的话"
2. **素材库建设**:用 `/clear-signing-test` 场景页截 20+ 场景图、录对比演示的原始素材、整理成可复用素材库
3. **度量看板**:接好 Cloudflare Web Analytics,建每周漏斗表(曝光→试用→激活代理指标→购买),她负责每周五填数
4. **VOC 挖矿**(leads 文档既定研究缺口):MetaMask/Rainbow/Coinbase Wallet 的 1-3 星差评 + r/ethereum 的 drained/seed phrase 帖,产出**带原话引用的 VOC 词库**(用户描述痛点的原始词汇,按画像归类)

**为什么 VOC 是新人的完美首战:** 零发布风险、强制她泡在真实用户语言里、产出直接喂给未来所有文案、且能验证画像假设(leads 文档明确标注画像是中置信度推理)。

**通过标准:** VOC 词库 ≥100 条原话引用且按画像/痛点归类;目录全部提交;看板连续两周有数。

### 阶段 3(第 5-8 周):带训发布 —— 有模板的内容,全部过审批窗口

从**模板化程度最高**的内容开始(判断量小,格式即护栏):

1. **每周对比视频**(计划 P2-5):选题从 leads 第一/二梯队清单里拿(#4/#17/#16/#5/#11/#1 轮换),格式固定(30 秒并排录屏),她写脚本+制作,你在审批窗口过一眼事实
2. **本地化内容**:把已审内容做多语言版本(14 语言包是现成杠杆)——翻译不产生新事实,风险天然低
3. **Reddit 答题(联合模式)**:前 4 条她起草、你逐句改;之后她发你抽查。透明利益申报句是固定模板
4. 官网落地页文案微调、商店 listing 文案初稿(上架前反正要过你)

**明确还不给她的:** Show HN / 安全事件评论 / 任何涉及价格变动、审计状态、竞品对比结论的原创长文。

**通过标准:** 连续 4 周发布节奏不断;审批窗口的事实修改次数趋近于零(这是"校准完成"的信号)。

### 阶段 4(第 9-12 周起):渐进放权 —— 她跑日常,你守边界

- 她**独立拥有**:每周内容节奏、Reddit、本地化、目录/素材维护、漏斗周报、社区常规问答
- 她**先发后备案**:已有模板的内容格式
- 仍需**你过目**:新内容形式的第一篇、任何触发 §三 红线清单的内容、安全事件评论(她 24h 内出初稿+你做技术事实核查后发——这条永久保留,因为错误的技术拆解会在 A 画像面前自杀)
- 第 12 周:她主持全量复盘(90 天漏斗基线),提出下季度渠道加减法,你只做决策

**毕业标准(可量化):** ① 连续 8 周节奏不断档;② 审批窗口连续 4 次零事实修正;③ 能独立完成一次事件评论初稿且技术拆解无硬伤;④ 漏斗周报能自己指出瓶颈层并给出下周分配建议。

## 二、职责分界(RACI,贴墙版)

| 事项 | 她 | 你 |
|---|---|---|
| 周常内容(视频/Reddit/本地化/目录) | **负责+执行** | 阶段3审批→阶段4抽查 |
| VOC/竞品研究、漏斗周报 | **负责+执行** | 每周会听 5 分钟 |
| 安全事件评论 | 起草(24h 模板) | **技术核查后才能发,永久** |
| Show HN / PH launch / 播客上台 | 协助准备 | **你出面**(单人创始人的脸就是信任资产) |
| 定价、促销、区域价变动 | 提建议 | **你决策**(pricing-analysis 护栏不可破) |
| 任何新的产品功能声明 | 先查事实库,查不到就问 | 你确认(或指给她代码/文档位置) |
| 广告投放、返利、KOL 付费 | ——— | **冻结项**(distribution-plan §四,谁都不做) |

## 三、发布前红线清单(她的每一次发布都跑一遍,打印贴屏幕边)

发布前自问,任何一条不确定 → 停,进审批窗口:

- [ ] **审计**:是否暗示"已审计/审计中/审计已排期"?(唯一合法表述:Safe 合约经审计;Vela 自身集成未经独立审计、无排期 —— whitepaper 原话)
- [ ] **免费**:是否出现 "free app"?(合法:free on web / no token;移动端永远是 $39.99 once)
- [ ] **代币**:是否有任何可被解读为发币/积分/空投暗示的表述?(绝对红线)
- [ ] **事实**:每一个产品声明能否在 CONTENT-SOURCE-100-CLUES.md / whitepaper / 代码里指出出处?(README 不算出处)
- [ ] **竞品**:是否点名贬损?(对比只做"机制对比"——盲签 vs 可读签名,不做"XX 钱包不安全"式论断)
- [ ] **收益**:是否有任何资产升值/收益暗示?(钱包不是投资产品)
- [ ] **"先用小额"**:讲安全时是否把它作为卖点而非免责声明带上了?
- [ ] **语气**:读起来像 why-we-charge.md 吗?(不感叹号、不 hype、敢说局限)

## 六、常见失败模式(提前打疫苗)

1. **节奏雪崩**:追热点发一堆→烧尽→断更一个月。解法:每周产能上限就是日历模板,超出的想法进 backlog
2. **数字虚荣**:追曝光量不追漏斗。解法:周会只讨论"瓶颈层",曝光暴涨但试用不动=白干
3. **过度承诺**:被社区追问时现场发明产品事实("会支持的!")。解法:标准回答话术——"好问题,我和创始人确认后回复你",没有例外
4. **模仿主流钱包营销**:空投预告/KOL 矩阵/价格喊单是这个品牌的毒药,不是参考对象
5. **害怕发布**:红线清单是让她敢发的工具,不是不发的理由——清单全过 = 放心发,错了也只是效果差,不是事故
