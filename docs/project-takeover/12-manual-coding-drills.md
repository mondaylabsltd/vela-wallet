# 12 — 无 AI 手工演练任务集 (Manual Coding Drills)

> 目的:把"AI 写码创始人"训练成能独立改+测+排障的接管者(14 号文档目标 L4/L5)。
> 基线 commit `73d7aac`(2026-07-02)。任务由浅入深,与 14 号文档训练单元 U0–U7 对齐;每个任务在**对应单元完成后**解锁。
>
> **本文件刻意不含任何实现方案、函数名、文件位置。**那些属于 H3–H5 级提示,按需由陪练另行给出。

## 通用规则(适用于全部任务)

1. **无 AI**:全程禁用 Claude/Copilot/ChatGPT 等一切 AI 辅助(包括 IDE 内联补全的 AI 部分)。允许:搜索引擎、MDN/Expo/Jest/Playwright 官方文档、本仓库内文档(除非任务单独禁用)。
2. **演练分支**:每个任务从 `main` 拉 `drill/<编号>` 分支开始,结束时 `git status` 干净(全部提交或全部丢弃),不合入 `main`。
3. **时间盒**:超预计用时 50% 仍无进展 → 暂停,申请下一级提示,并在完成记录中如实记录提示等级。
4. **提示等级**(记录用):H0 无提示 / H1 澄清需求 / H2 指向某号交接文档的章节 / H3 文件位置 / H4 函数或符号名 / H5 实现思路。H3 及以上意味着该任务未通过,需安排复测。
5. **口述验收**:凡验收标准含"口述"项,由陪练(人或另开的 AI 会话,仅限验收环节)按 03/04/06 号文档核对,答错关键点即不通过。
6. 每个任务完成后,把结果写入本文件末尾"完成记录"表,并更新 14 号文档训练日志。

## 任务 D-01:定位一个功能入口(代码地图)

- **上下文边界**:只允许看 `src/` 全目录与 `package.json`;**禁止**在本任务期间翻阅 docs/project-takeover/(那是答案卷)。
- **限制条件**:只读,不改任何代码;可以用 grep/IDE 跳转,可以本地起 dev 服务加临时 console.log 验证(结束前删掉)。
- **验收标准**:
  - 调用链笔记与 03 号文档第 2 节核对,关键跳转(≥5 跳)全部命中,行号误差 ≤10 行;
  - 现场演示:在链路中任选一跳加日志,跑起 App 在 parallel space 触发一次发送,日志按预期顺序打印;
